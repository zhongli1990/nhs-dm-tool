# Release Notes v0.2.2

Date: 2026-02-27

## Summary

v0.2.2 is the enterprise Dockerization and integration release for the NHS PAS data-migration platform, merging the 0.2.1 UI/backend feature line into the current full-stack runtime architecture.

## Key updates

1. Enterprise container architecture
- Introduced a full Docker Compose stack for frontend, backend, nginx, and PostgreSQL.
- Standardized service ports to enterprise-safe host range `9133-9139`.
- Added production compose profile and dedicated Dockerfiles for backend/frontend/nginx.

2. PostgreSQL service integration
- Added PostgreSQL 16 containerized runtime for full-stack operation.
- Added health-checked startup ordering and service dependency wiring.
- Added environment contract for backend DB connectivity.

3. 0.2.1 feature merge into current mainline
- Merged backend `v0.2.1` API capabilities into current services layout:
  - `GET /api/meta/version`
  - `POST /api/connectors/preview`
- Added runtime version manifest support via `services/version_manifest.json`.
- Merged frontend version visibility and connector preview UX improvements.

4. Enterprise licensing and IP clarity
- Added enterprise license document with IP and copyright ownership:
  - Owner: Lightweight Integration Limited
  - Contact: Zhong@li-ai.co.uk

5. Deployment strategy hardening
- Preserved Windows Server non-Docker deployment path (Python venv/manual runtime) while adopting Docker-first operations.
- Updated project structure and documentation for dual deployment models.

## Runtime verification

- Docker images built successfully for backend/frontend.
- Compose stack recreated successfully:
  - `dmm-frontend` on `9133`
  - `dmm-backend` on `9134`
  - `dmm-nginx` on `9135`
  - `dmm-postgres` on `9136`
- Health checks confirmed backend and postgres healthy after restart.
